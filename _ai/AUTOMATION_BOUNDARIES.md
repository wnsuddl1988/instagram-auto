# Automation Boundaries

## Allowed Automation

minimum necessary HANDOFF/report/review/NEXT_ACTION/PROJECT_STATE/TRACKING updates, targeted test, diff summary, commit message draft, Owner-approved checkpoint commit, reusable Obsidian/knowledge update only when state actually changed

## Forbidden Without Explicit Approval

push without explicit Owner approval, deploy, DB schema/migration, env/secret, dependency, external sending, payment/purchase/sale/order, investment/trading execution, legal execution, account/customer/production data changes; commit without Owner approval or explicit Codex handoff authorization
