# AutoShorts AI 간단 실행 매뉴얼

이 문서는 Owner가 실제로 프로그램을 쓸 때 보는 짧은 실행 순서입니다.
**터미널 명령을 외울 필요 없이, 웹 화면에서 버튼을 눌러 진행합니다.**

## 1. 프로그램 열기

프로젝트 폴더에서 한 번만 실행합니다.

```powershell
cd C:\Users\PC\jjy\instagram-auto
pnpm dev
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:3000/money-shorts
```

> **중요**
> 실제 영상 생성과 게시 준비는 Owner PC에서 `pnpm dev`로 연 **로컬 화면**에서 실행합니다.
> 배포된 웹사이트는 상태 확인과 화면 검토용입니다. 배포 사이트에서 버튼을 누르면
> "로컬 화면에서만 동작합니다"라는 안내가 나옵니다.

## 2. 자동 쇼츠 만들기 (메인 흐름)

`/money-shorts` 맨 위의 **"자동 쇼츠 만들기"**에서 1번부터 순서대로 누르면 됩니다.

| 순서 | 단계 | 하는 일 |
|---|---|---|
| 1 | `카테고리 선택` | 8개 카테고리 중 하나 선택 (전부 주제 추천 가능) |
| 2 | `주제 추천받기` | **누를 때마다 새 주제 묶음**을 만듦. [다른 주제 보기]를 누르면 **최근에 본 주제는 빼고** 새로 만들어서, 순서만 바뀌는 일이 없음. 재테크팁 제목은 **월급날·배달앱·구독·카드값·장바구니 같은 실제 소비 행동**이 들어가, 보자마자 "이거 내 얘기잖아" 싶게 나옴. **로컬 품질 평가기가 후보를 점수화해 약한 건 걸러내거나 자동으로 다듬어**, 통과한 상위 후보만 화면에 옴(주제 옆 `품질 NN`/`다듬음` 배지) |
| 3 | `대본 만들기` | 고른 주제로 **실제 읽히는 대본**(음성·영상이 그대로 사용) + **대본 품질 점수**(좋은 이유·고친 부분·주의할 점) + **첫 3초 훅** + **영상 자막 6개** + **장면 그림 계획**(보조) + **SNS 설명글 초안**을 생성. 내부적으로 **강한 후킹형·공감형·반전형 3개 후보**를 만들어 **최고점 1개**를 기본으로 보여줌. 대본 첫 3문장은 `구체 행동 → 그때 마음 → 돈 새는 지점` 순서라 훈계가 아니라 "들킨 느낌"으로 시작함 |
| 4 | `음성 만들기` | 고른 대본 기준 테스트용 소리 생성 (실제 목소리는 다음 승인 필요) |
| 5 | `영상 만들기` | 고른 주제의 대본·자막이 들어간 시안 mp4를 약 10초 만에 생성 (업로드 없음) |
| 6 | `미리보기` | 방금 만든 그 주제의 시안 영상을 화면에서 바로 재생 |
| 7 | `게시 전 점검` | 그 주제 영상 기준으로 키·중복·파일을 확인만 함 |
| 8 | `실제 업로드` | 확인 2개 체크 + **“업로드” 입력** 후 버튼을 눌러야만 실제 게시. 게시 기록이 남아 같은 콘텐츠는 다시 안 올라감 |

1~7단계는 **Owner PC 안에서만** 동작하고 아무것도 업로드하지 않습니다. 8단계만 실제 계정에 게시하며, 확인 절차 없이는 버튼이 켜지지 않습니다.

## 2-1. 세부 점검 도구 (고급)

같은 화면 아래 **"세부 점검 도구 (고급)"**에서 항목별 점검을 할 수 있습니다.

| 순서 | 버튼 | 하는 일 | 정상 결과 |
|---|---|---|---|
| 1 | `상태 확인` | 영상 파일과 설정이 갖춰졌는지 확인 | 「시스템 준비 상태가 정상입니다」 |
| 2 | `업로드 키 확인` | 업로드에 필요한 키 6개가 있는지 확인 | 「업로드 키 6개가 모두 준비되어 있습니다」 |
| 3 | `재업로드 차단 확인` | 이미 올린 영상이 다시 안 올라가는지 확인 | 「이미 올린 샘플이라 재업로드가 안전하게 차단됐습니다」 |
| 4 | `게시 전 점검` | 새 콘텐츠가 올릴 준비가 됐는지만 확인 | 「게시 전 점검만 완료했습니다. 실제 업로드는 실행하지 않았습니다」 |
| 5 | `실제 업로드` | 이 점검 화면에서는 실행 안 됨 | 위 자동 만들기 8단계의 확인 절차에서만 실행 |

각 버튼은 **확인만 하고 업로드하지 않습니다.**
결과는 사람이 읽는 한 줄 요약으로 먼저 보이고, 자세한 내용은 「개발자용 자세한 내용 보기」를 눌러야 나옵니다.

### 키 값은 화면에 나오지 않습니다

`업로드 키 확인`은 키가 **있는지 없는지만** 알려줍니다.
키 값, 길이, 앞자리, hash는 화면에도 결과 파일에도 절대 나오지 않습니다.

### 3번이 "차단됐습니다"로 나오는 게 정상입니다

이미 올린 샘플을 실수로 다시 올리지 않도록 막는 장치가 작동한다는 뜻입니다.
실패가 아니라 **안전장치가 잘 켜져 있다는 확인**입니다.

## 3. 새 콘텐츠 게시 전 점검 (4번 버튼)

4번 버튼은 경로 3개를 입력해야 활성화됩니다.

| 입력칸 | 넣을 것 |
|---|---|
| 콘텐츠 정보 파일 경로 | 새 콘텐츠 정보 JSON (`dual_platform_content_unit...json`) |
| 업로드 기록 파일 경로 | 이미 올린 콘텐츠 기록 JSON (`publish-ledger.json`) |
| 결과 저장 폴더 경로 | 결과를 저장할 폴더 (`C:\tmp\money-shorts-os\...`) |

경로는 **전체 경로**로 입력합니다. 세 칸을 모두 채우면 버튼이 켜집니다.

이 버튼을 눌러도 **인스타그램·유튜브에 아무것도 올라가지 않습니다.**
올릴 준비가 됐는지만 확인하고, 결과 파일을 지정한 폴더에 남깁니다.

막힌 경우 화면이 이유를 한국어로 알려줍니다. 예:

- 「이미 올린 콘텐츠라 재업로드가 안전하게 차단됐습니다.」
- 「영상 파일을 찾을 수 없습니다.」
- 「업로드 키가 준비되지 않아 점검을 마치지 못했습니다.」

## 4. 실제 업로드 (확인 게이트)

8번 `실제 업로드` 버튼은 **확인 절차를 모두 마쳐야만 켜집니다.** 켜지기 전까지는 인스타그램 게시·유튜브 업로드가 일어나지 않습니다.

버튼이 켜지는 조건(모두 충족해야 함):

1. `영상 만들기`로 시안 영상을 만들었을 것
2. `게시 전 점검`을 통과했을 것
3. 확인 체크 2개를 켰을 것 (미리보기 검토 / 실제 계정 게시 동의)
4. 입력칸에 **`업로드`** 를 직접 타이핑했을 것

이 4가지가 모두 충족되면 버튼이 활성화되고, 누르면 실제 인스타그램·유튜브 계정에 게시됩니다. 서버도 같은 조건을 다시 검사하며, 어느 하나라도 빠지면 한국어로 막습니다.

> **주의**: 이 업로드는 **Owner PC의 로컬 `pnpm dev` 화면에서만** 가능합니다. 배포(Vercel) 사이트는 로컬 영상 파일에 접근할 수 없어 상태 확인·안내만 됩니다. 이미 게시된 콘텐츠는 게시 기록으로 자동 차단되어 다시 올라가지 않습니다.

## 5. 업로드 문구 원칙

영상만 올리면 의미가 없으므로, 매번 아래를 지킵니다.

- 첫 줄은 바로 이해되는 hook으로 작성
- 제목은 짧고 검색 의도가 분명하게 작성
- 설명에는 핵심 요약 + 저장/팔로우 유도 문구 포함
- 해시태그는 관련 있는 것만 8~12개 사용
- 무관한 유행 태그, 낚시 태그, 과장 문구 금지

## 6. 절대 하지 말 것

- 이미 올린 `t1_lifestyle_inflation` 샘플 재업로드 금지
- 같은 `contentId/version` 재사용 금지
- 같은 `out-dir`로 재실행 금지
- `.env.local` 내용을 채팅에 붙여넣기 금지
- 업로드 실패 후 ledger를 수동으로 성공 처리 금지

## 7. 막혔을 때 Codex에게 줄 것

문제가 생기면 secret 값은 빼고 아래만 알려주면 됩니다.

- 누른 버튼 이름
- 화면에 나온 요약 문구
- 「개발자용 자세한 내용 보기」의 `status` 또는 `blockerCode`
- 인스타그램/유튜브 중 어디까지 진행됐는지

## 부록. 터미널로 확인하고 싶을 때 (선택)

웹 화면이면 충분하지만, 터미널에서 같은 확인을 하려면 아래를 씁니다.

```powershell
# 업로드 키 확인 (값은 출력되지 않음)
pnpm owner:credential-preflight:local

# 준비 상태 + 재업로드 차단 확인
pnpm owner:ready-preflight
pnpm owner:ready-duplicate-guard-check
```

새 콘텐츠 게시 전 점검(업로드 안 함):

```powershell
node scripts/run-owner-command-with-local-env-no-log.mjs final-e2e-publish `
  --content-unit <manifest.json> --ledger <publish-ledger.json> --out-dir <C:\tmp\money-shorts-os\작업폴더>
```

정상이면 `PREFLIGHT_ONLY_OK`가 나옵니다. 외부 업로드는 하지 않습니다.

실제 업로드(`--arm`)는 **Owner의 별도 승인이 있을 때만** 실행합니다.
